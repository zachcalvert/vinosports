# 2026 World Cup

> Status: **In progress**


## Context
The 2026 FIFA World Cup (June 11 – July 19, hosted across USA/Mexico/Canada) is a natural addition to vinosports. This plan adds a worldcup/ league with full feature parity to EPL — matches, betting, parlays, futures, bots, discussions, activity, challenges. The tournament uses a new 48-team format (12 groups of 4, knockout round of 32 through final).

- Data source: football-data.org (free tier, competition code WC, 10 req/min)
- URL: vinosports.com/worldcup/
- Odds format: Decimal 1X2 (same as EPL — it's football)
- Settlement rule: Bets settle on 90-minute result only (standard industry practice for 1X2)


## Phase 1: Package Skeleton + Core Models
Create worldcup/ with sub-apps, register in config, define match models, run migrations.

- **Key models**: worldcup/matches/models.py
- **Team** — Like EPL but adds confederation (AFC/CAF/CONCACAF/CONMEBOL/OFC/UEFA), country_code. Drops venue (venues are per-match in a World Cup).
- **Group** (new, no EPL equivalent) — letter (A-L, unique), teams M2M to Team.
- **Stage** (new) — name, stage_type choices (GROUP, ROUND_OF_32, ROUND_OF_16, QUARTER, SEMI, THIRD_PLACE, FINAL), order int for sorting.
- **Match** — Like EPL but adds:
  - FK to Stage, nullable FK to Group (only group matches)
  - venue, city fields (multi-country tournament)
  - Extra time scores: home_score_et, away_score_et
  - Penalty scores: home_score_penalties, away_score_penalties
  - Additional Status choices: EXTRA_TIME, PENALTY_SHOOTOUT
  - Properties: is_knockout, winner (resolves ET/pens for knockouts)
- **Standing** — Per-group (FK to Group + team, unique together) instead of per-season.
- **Odds** — Same as EPL: match FK, bookmaker, home_win/draw/away_win decimal. Lives in matches app (following EPL pattern at epl/matches/models.py:195).
- **Config modifications** (8 existing files)

## Phase 2: Data Client + Seed Command
Build WorldCupDataClient targeting football-data.org v4 API, sync functions, offline JSON bundles, and seed_worldcup management command.

`worldcup/matches/services.py`
Reference: epl/matches/services.py (BDL client pattern)

- **Base URL**: https://api.football-data.org/v4
- **Auth**: X-Auth-Token header
- **Rate limiting**: 6-second throttle between requests (10 req/min limit)
- **Endpoints**: /competitions/WC/teams, /competitions/WC/matches, /competitions/WC/standings
- **Offline mode**: load from worldcup/matches/static_data/*.json

**Sync functions**: `sync_stages() → sync_teams() → sync_groups() → sync_matches() → sync_standings()`

- `worldcup/website/management/commands/seed_worldcup.py`
- **Orchestrator**: `stages → teams → groups → matches → standings → odds generation. Flags: --offline, --skip-odds`.

## Phase 3: Betting Models + Odds Engine
`worldcup/betting/models.py`
Reference: epl/betting/models.py

- **BetSlip**: Selection HOME_WIN/DRAW/AWAY_WIN, FK to Match, decimal odds
- **Parlay/ParlayLeg**: Same as EPL, decimal combined_odds
- **FuturesMarket**: MarketType choices = WINNER, FINALIST, GROUP_WINNER. Adds nullable group FK for GROUP_WINNER markets
- **FuturesOutcome/FuturesBet**: Same as EPL

**Odds + settlement**
- worldcup/betting/odds_engine.py — Adapted from epl/betting/odds_engine.py. Team strength from group standings + seed rankings
- worldcup/betting/futures_odds_engine.py — Winner (48-team), group winners (4-team per group), finalist
- Settlement: 90-minute result only. Knockout 1-1 after 90 min = DRAW bet wins
